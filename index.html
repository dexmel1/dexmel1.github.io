> DungeonExplorer Overview

DungeonExplorer is a dungeon-crawling adventure game where players explore interconnected rooms, gather legendary items, and ultimately face a final boss in the Dungeon. The original version was a Python console game relying on linear procedural logic and global state. The capstone version is a complete re-architecture into a C# WPF MVVM application with a graphical interface, persistent scoring, advanced pathfinding algorithms, and a dynamic state machine.

> Original Python Console Game

The Python version used simple dictionaries and a command-driven loop. It functioned well as an academic exercise but lacked separation of concerns, formal data structures, or scalability. Key limitations included:

  • Procedural structure with tightly coupled input and game logic
  • No abstraction for rooms, items, or player state
  • Difficult to extend or refactor for new interfaces

> C# MVVM Conversion

The enhanced artifact is a full C# WPF application implementing MVVM principles. The game logic resides in a GameService and GameViewModel, while the UI is defined strictly through data binding. This redesign allowed:

  • Clear separation of game logic, presentation, and state
  • Reusable domain models for rooms, players, and items
  • Event-driven updates instead of a procedural game loop

> Artifact Code Review

The initial code review documents the transformation from the Python prototype to the C# MVVM architecture, including refactoring decisions, UI design choices, and identified areas for algorithmic and database-driven enhancements.

> Enhancement Overview

DungeonExplorer was enhanced across three major dimensions:

  • Software Design & Engineering: Full MVVM restructuring, modular class design, clear state machine
  • Algorithms & Data Structures: BFS shortest paths, Held–Karp optimal item routing, randomized but solvable item placement
  • Data Persistence: SQLite leaderboard, EF Core integration, score filtering, player identity tracking

>> Software Design & Engineering

The move to MVVM required designing a maintainable, layered architecture. Key engineering upgrades include:

  • Replacing procedural logic with object-oriented models
  • Creating a dedicated GameService for game rules and transitions
  • Implementing a ViewModel that exposes bindable properties and commands
  • Using state patterns to manage menu, gameplay, and game-over transitions

>> Algorithms & Data Structures

Milestone Three focused on algorithmic depth:

  • BFS for computing all shortest paths between rooms
  • Held–Karp dynamic programming for optimal multi-item routing
  • Par scoring system based on algorithmically evaluated efficiency
  • Randomized item distribution validated for reachability via BFS

>> Data Persistence (SQLite + EF Core)

Milestone Four added a full persistence layer using SQLite and Entity Framework Core:

  • Relational schema for players and scores
  • Leaderboard with score history, par, win/loss, and items collected
  • Filters for “My Scores Only” and “Wins Only”
  • MVVM-compatible LeaderboardService for clean data access
