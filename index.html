> The DungeonExplorer Artifact

DungeonExplorer began life as a text-based adventure written in Python. The original game let the player move through a small dungeon map, collect items, and face a final encounter that resulted in either a win or a loss. All interaction happened through the console using commands like go north and get key.

For the capstone, I selected this artifact because it is small enough to understand quickly, but rich enough to demonstrate software design, refactoring, and UI transformation. The enhanced version is a C# GUI application that preserves the core gameplay loop while significantly improving structure, readability, and player experience.

> Original Python Console Game

The original Python implementation used a global state dictionary to represent rooms, items, and player location. A main game loop repeatedly printed the current room, read the player’s text input, and then updated state based on a series of conditional checks. While functional, the design had several limitations:

  • Heavily procedural with limited separation of concerns
  • Game logic and user interface (console I/O) tightly coupled
  • Difficult to extend with new features or alternate interfaces

> C# GUI Conversion

The enhanced DungeonExplorer is implemented in C# with a graphical user interface. Rather than typing commands, the player interacts with buttons for movement, item interaction, and game actions. The underlying codebase was restructured to follow a more modern architecture that separates the UI from the game logic.

The core goals of the conversion were:

  • Separate game rules from presentation logic
  • Introduce a clear domain model for rooms, items, and the player
  • Leverage the C# ecosystem and tooling (Visual Studio, .NET, etc.)

> Artifact Code Review

In this code review, I walk through the evolution of DungeonExplorer, from the original Python console version to the refactored C# GUI implementation. I focus on design decisions, trade-offs, and how applying patterns like MVVM improved testability and maintainability.

> Enhancement Overview

The primary enhancement focus for DungeonExplorer is software design and engineering, specifically refactoring an academic prototype into a structured application that resembles professional game architecture. Along the way, I also improved the way game data is represented and explored future options for persistence.

>> Software Design & Engineering

The C# version of DungeonExplorer is organized around clear responsibilities and a more maintainable structure. Key changes include:

  • MVVM-style separation: A GameViewModel exposes properties such as CurrentRoom, AvailableMoves, and Inventory, while the view handles only layout and binding.
  • Domain classes: Separate classes for Room, Item, and Player replace loosely defined dictionaries and global variables.
  • Command-based input: Movement and item actions are encapsulated as commands, which makes it easier to wire buttons and potentially add keyboard shortcuts later.

These changes demonstrate the ability to migrate a prototype into a layered architecture that is easier to extend and reason about, aligning with industry expectations for software design and engineering.

>> Algorithms & Sata Structures

Although DungeonExplorer is a relatively small game, it still benefits from intentional data structures and algorithm choices:

  • The dungeon is modeled as a graph of Room nodes, each with references to adjacent rooms (north, south, east, west). This makes it straightforward to validate legal moves and visualize the map.
  • The player’s inventory is stored as a collection (e.g., List<Item>) that drives both UI binding and win-condition logic.
  • Win/lose conditions are evaluated through a small rule engine that checks both player location and inventory contents, keeping the game loop simple and deterministic.

In a larger production setting, this same approach could be extended to pathfinding, quest tracking, or more advanced combat systems.

>> Data Persistence

For DungeonExplorer, persistence is intentionally lightweight.

While this artifact does not focus on a full relational database, the same architectural choices would make it straightforward to add features like:

  • Saving and loading player progress
  • Tracking high scores across multiple runs

> The Final Build

The final DungeonExplorer build is a C# GUI application that preserves the spirit of the original Python console game while showcasing improved structure, user experience, and readiness for future enhancements. Players can visually explore the dungeon, monitor their inventory at a glance, and clearly see their path to victory or defeat.
